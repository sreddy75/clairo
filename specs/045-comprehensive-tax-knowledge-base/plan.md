# Implementation Plan: Comprehensive Australian Tax Knowledge Base

**Branch**: `045-comprehensive-tax-knowledge-base` | **Date**: 2026-03-05 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/045-comprehensive-tax-knowledge-base/spec.md`

## Summary

Build a comprehensive Australian tax knowledge base by ingesting publicly available legislation, ATO rulings, case law, and practitioner guidance into Clairo's existing Pinecone-based RAG infrastructure. Extends spec 012 (Knowledge Base) with structure-aware chunking for legal content, hybrid search (semantic + BM25), cross-encoder re-ranking, legal query routing, and post-generation citation verification. The result replaces Tax Guru's $4,500 knowledge product with a superior, client-contextual alternative.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript 5.x (frontend)
**Primary Dependencies**: FastAPI, SQLAlchemy 2.x, Pydantic v2, Pinecone, Voyage API (embeddings), anthropic SDK, Celery + Redis, BeautifulSoup4, rank-bm25 (NEW), sentence-transformers (NEW)
**Storage**: PostgreSQL 16 (metadata, cross-references), Pinecone Serverless (vectors, ~100K chunks)
**Testing**: pytest with pytest-asyncio, httpx for API testing, factory_boy for fixtures
**Target Platform**: Linux server (Railway), Vercel (frontend)
**Project Type**: Web application (backend + frontend)
**Performance Goals**: Retrieval + re-ranking < 500ms p95, full response (with LLM) < 3s p95
**Constraints**: 10-second crawl delay for legislation.gov.au, ATO rate limiting (polite crawling), ~100K vector budget on Pinecone Serverless
**Scale/Scope**: ~80,000-100,000 content chunks, 7+ content sources, 9+ specialist domains

## Phase 0: Research

Research is already complete in `research.md`. Key decisions:

| Decision | Rationale | Alternatives Rejected |
|----------|-----------|----------------------|
| Pinecone (keep existing) | Already in production, supports sparse vectors for hybrid search | Qdrant self-hosted (migration cost, data residency less critical for public content) |
| Voyage 3.5 lite (keep, evaluate later) | In production, ~78-80% NDCG. Phase 10 will benchmark Kanon 2 (86%) | Immediate switch (re-embedding cost + risk without benchmarking) |
| EPUB for legislation | Structured HTML inside ZIP, best for section-level parsing | PDF (binary, hard to parse), DOCX (less structured) |
| BM25 via rank-bm25 library | Simple Python library, no infra changes. Fuse with Pinecone results | Pinecone sparse vectors (requires plan upgrade check), Elasticsearch (new infra) |
| cross-encoder/ms-marco-MiniLM-L-6-v2 | 15-25% precision improvement, fast enough (~100ms), well-studied | Cohere Rerank (commercial dependency), no re-ranker (lower precision) |
| Open Australian Legal Corpus | CC-BY 4.0, 232K docs, includes Federal Court + High Court | AustLII (prohibits AI use), direct court scraping only (slower, smaller corpus) |
| Section-level chunking for legislation | Natural atomic unit, preserves legal meaning | Arbitrary character splits (destroys legal meaning) |
| Post-generation citation verification | Architectural hallucination prevention | Manual curation (Tax Guru approach, doesn't scale) |

## Phase 1: Design

### Data Model

See `data-model.md` for complete schema.

Key additions:
- `LegislationSection` — tracks ingested legislation for cross-referencing
- `ContentCrossReference` — section-to-section links for graph traversal
- `TaxDomain` — specialist domain configuration
- Extended `ContentChunk` metadata (content_type, section_ref, cross_references, topic_tags, fy_applicable, court, case_citation)

### API Contracts

See `contracts/` directory for OpenAPI schemas.

Key new endpoints:
- `GET /api/v1/knowledge/domains` — list specialist tax domains
- `POST /api/v1/knowledge/search` — enhanced hybrid search
- `GET /api/v1/knowledge/legislation/{section_ref}` — specific section lookup
- `POST /api/v1/admin/knowledge/ingest/legislation` — trigger legislation sync
- `POST /api/v1/admin/knowledge/ingest/case-law` — trigger case law ingestion
- `GET /api/v1/admin/knowledge/freshness` — content freshness report

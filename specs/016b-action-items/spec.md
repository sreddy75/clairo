## Overview

Convert AI-generated insights into curated, actionable work items with human-in-the-loop review. This transforms Clairo from an alert system into an AI-powered practice management tool.

## Goals

1. **Human-in-the-loop curation** - AI suggests, accountant decides what becomes work
2. **Team collaboration** - Assign tasks to team members
3. **Prioritized work queue** - Clear view of what needs attention
4. **Traceability** - Link actions back to source insights
5. **Accountability** - Track completion and deadlines

## Data Model

### ActionItem (New Model)

```python
class ActionItemStatus(str, Enum):
    PENDING = "pending"        # Created, not started
    IN_PROGRESS = "in_progress" # Being worked on
    COMPLETED = "completed"    # Done
    CANCELLED = "cancelled"    # No longer needed

class ActionItemPriority(str, Enum):
    URGENT = "urgent"          # Do today
    HIGH = "high"              # This week
    MEDIUM = "medium"          # This month
    LOW = "low"                # When possible

class ActionItem(Base):
    __tablename__ = "action_items"

    id: UUID
    tenant_id: UUID                    # RLS

    # Content
    title: str                         # Task title (can be edited from insight)
    description: str | None            # Additional notes

    # Source (optional - can create standalone tasks too)
    source_insight_id: UUID | None     # FK to insights table

    # Client context
    client_id: UUID | None             # FK to xero_connections
    client_name: str | None            # Denormalized for display

    # Assignment & scheduling
    assigned_to_user_id: str | None    # Clerk user ID
    assigned_to_name: str | None       # Denormalized
    assigned_by_user_id: str           # Who created/assigned
    due_date: date | None              # Deadline
    priority: ActionItemPriority

    # Status tracking
    status: ActionItemStatus
    created_at: datetime
    updated_at: datetime
    started_at: datetime | None
    completed_at: datetime | None

    # Completion notes
    resolution_notes: str | None       # What was done
```

### Relationships

```
Insight (1) ──────▶ (0..1) ActionItem
   │                        │
   │ source_insight_id ◀────┘
   │
   └── Can exist without action item (dismissed/resolved directly)

ActionItem can also be standalone (no source insight)
   └── For custom tasks not generated by AI
```

## Features

### 1. Convert Insight to Action Item

From the Insight detail view or card:
- "Create Action Item" button
- Pre-fills: title (from insight), client, priority (from insight priority)
- User can edit: title, description, due date, assignee

### 2. Action Items List Page

**Route**: `/action-items`

**Views**:
- **My Items**: Assigned to current user
- **Team Items**: All items for the tenant (managers)
- **By Client**: Grouped by client

**Filters**:
- Status (pending, in progress, completed)
- Priority (urgent, high, medium, low)
- Assignee
- Client
- Due date range

**Sorting**:
- Due date (default)
- Priority
- Created date
- Client name

### 3. Action Item Card

Display:
- Title
- Priority indicator (color-coded)
- Client name (if applicable)
- Due date (with overdue highlighting)
- Assignee avatar/name
- Status badge
- Source insight link (if from insight)

Actions:
- Mark as started
- Mark as complete
- Edit details
- Reassign
- Change priority
- Delete/cancel

### 4. Quick Actions from Insights

On Insight cards/detail:
- "Convert to Action" - creates action item
- Shows existing action item if already converted
- "View Action Item" link

### 5. Dashboard Widget

Small widget showing:
- Count of pending action items
- Overdue items (highlighted)
- Next 3 due items
- Link to full action items page

### 6. Team Assignment

If team members exist (from Clerk):
- Dropdown to assign to team member
- Email notification on assignment (optional, Phase 2)
- Filter by "assigned to me" vs "team"

## API Endpoints

```
POST   /api/v1/action-items              # Create action item
GET    /api/v1/action-items              # List with filters
GET    /api/v1/action-items/:id          # Get single item
PATCH  /api/v1/action-items/:id          # Update item
DELETE /api/v1/action-items/:id          # Delete item

POST   /api/v1/action-items/:id/start    # Mark as in progress
POST   /api/v1/action-items/:id/complete # Mark as complete
POST   /api/v1/action-items/:id/assign   # Reassign to user

# Convenience endpoint
POST   /api/v1/insights/:id/convert-to-action  # Create action from insight
```

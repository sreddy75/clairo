## New Entities

### `tax_plans`

A tax planning session for a specific client and financial year. One plan per client per FY. Contains base financials (from Xero or manual entry), calculated tax position, and status tracking.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, default `uuid4` | Primary key |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL, INDEX | Multi-tenancy isolation |
| `xpm_client_id` | `UUID` | FK → `xpm_clients.id`, NOT NULL | The practice client this plan is for |
| `xero_connection_id` | `UUID` | FK → `xero_connections.id`, NULL | Xero org link (null if manual entry only) |
| `financial_year` | `String(10)` | NOT NULL | e.g. "2025-26" |
| `entity_type` | `Enum(EntityType)` | NOT NULL | company, individual, trust, partnership |
| `status` | `Enum(TaxPlanStatus)` | NOT NULL, default `draft` | Plan lifecycle status |
| `data_source` | `Enum(DataSource)` | NOT NULL | xero, manual, xero_with_adjustments |
| `financials_data` | `JSONB` | NOT NULL | Income, expenses, credits breakdown (see schema below) |
| `tax_position` | `JSONB` | NULL | Calculated base tax position (null until calculated) |
| `notes` | `Text` | NULL | Accountant's free-text notes |
| `xero_report_fetched_at` | `DateTime(tz)` | NULL | When Xero P&L was last pulled |
| `created_at` | `DateTime(tz)` | NOT NULL, default `now()` | |
| `updated_at` | `DateTime(tz)` | NOT NULL, default `now()` | |

**Unique Constraint**: `uq_tax_plan_client_fy` on `(xpm_client_id, financial_year)` — enforces one plan per client per FY.

**Indexes**:
- `ix_tax_plans_tenant_id` on `(tenant_id)` — RLS and listing queries
- `ix_tax_plans_tenant_status` on `(tenant_id, status)` — filtered listing
- `ix_tax_plans_xpm_client_id` on `(xpm_client_id)` — client-scoped lookups

### `tax_scenarios`

A modelled what-if scenario within a tax plan. Generated by the AI agent, each scenario shows the impact of a specific strategy on the client's tax position.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, default `uuid4` | Primary key |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL, INDEX | Multi-tenancy isolation |
| `tax_plan_id` | `UUID` | FK → `tax_plans.id` ON DELETE CASCADE, NOT NULL | Parent plan |
| `title` | `String(255)` | NOT NULL | Short description, e.g. "Prepay 6 months rent" |
| `description` | `Text` | NOT NULL | Full AI-generated description of the strategy |
| `assumptions` | `JSONB` | NOT NULL, default `{}` | Key assumptions made in the model |
| `impact_data` | `JSONB` | NOT NULL | Before/after tax impact (see schema below) |
| `risk_rating` | `Enum(RiskRating)` | NOT NULL | conservative, moderate, aggressive |
| `compliance_notes` | `Text` | NULL | Compliance warnings, ATO references |
| `cash_flow_impact` | `Decimal(15,2)` | NULL | Net cash flow change |
| `sort_order` | `Integer` | NOT NULL, default `0` | Display ordering within the plan |
| `created_at` | `DateTime(tz)` | NOT NULL, default `now()` | |

**Indexes**:
- `ix_tax_scenarios_tax_plan_id` on `(tax_plan_id)` — listing scenarios for a plan
- `ix_tax_scenarios_tenant_id` on `(tenant_id)` — RLS

**`impact_data` JSONB Schema**:
```json
{
  "before": {
    "taxable_income": 162000.00,
    "tax_payable": 40500.00,
    "net_position": 15500.00
  },
  "after": {
    "taxable_income": 132000.00,
    "tax_payable": 33000.00,
    "net_position": -7500.00
  },
  "change": {
    "taxable_income_change": -30000.00,
    "tax_saving": 7500.00,
    "net_benefit": 7500.00
  }
}
```

**`assumptions` JSONB Example**:
```json
{
  "items": [
    "Rent prepayment covers 6 months (July-December 2026)",
    "Prepayment made before 30 June 2026",
    "Qualifies under 12-month prepayment rule (s82KZM ITAA 1997)"
  ]
}
```

### `tax_plan_messages`

Conversation messages for a tax plan's AI chat session. Preserves full history for context-aware follow-ups and plan resumption.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, default `uuid4` | Primary key |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL, INDEX | Multi-tenancy isolation |
| `tax_plan_id` | `UUID` | FK → `tax_plans.id` ON DELETE CASCADE, NOT NULL | Parent plan |
| `role` | `String(20)` | NOT NULL | "user" or "assistant" |
| `content` | `Text` | NOT NULL | Message text content |
| `scenario_ids` | `ARRAY(UUID)` | NOT NULL, default `{}` | Scenarios generated/referenced by this message |
| `token_count` | `Integer` | NULL | Estimated token count for context management |
| `metadata` | `JSONB` | NOT NULL, default `{}` | Processing metadata (model, latency, etc.) |
| `created_at` | `DateTime(tz)` | NOT NULL, default `now()` | |

**Indexes**:
- `ix_tax_plan_messages_plan_id_created` on `(tax_plan_id, created_at)` — conversation history in order
- `ix_tax_plan_messages_tenant_id` on `(tenant_id)` — RLS

### `tax_rate_configs`

Australian tax rates, thresholds, and offsets stored as configuration data. One record per financial year per entity type (plus shared records for cross-entity rates like Medicare).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, default `uuid4` | Primary key |
| `financial_year` | `String(10)` | NOT NULL | e.g. "2025-26" |
| `rate_type` | `String(50)` | NOT NULL | individual, company, trust, partnership, medicare, lito, help |
| `rates_data` | `JSONB` | NOT NULL | Rate structure (varies by type, see below) |
| `effective_from` | `Date` | NOT NULL | When these rates take effect |
| `notes` | `Text` | NULL | Legislative reference or change notes |
| `created_at` | `DateTime(tz)` | NOT NULL, default `now()` | |
| `updated_at` | `DateTime(tz)` | NOT NULL, default `now()` | |

**Unique Constraint**: `uq_tax_rate_config_year_type` on `(financial_year, rate_type)`.

**Indexes**:
- `ix_tax_rate_configs_financial_year` on `(financial_year)` — lookup by FY

## State Transitions

### TaxPlan Status
```
draft → in_progress       (accountant starts working — first scenario created or financials modified)
in_progress → finalised   (accountant marks plan as complete)
finalised → in_progress   (accountant reopens for revision)
```

### Plan Replacement (FR-008)
```
When creating a new plan for the same client+FY:
  1. Confirm with user (frontend prompt)
  2. Delete existing plan (CASCADE deletes scenarios + messages)
  3. Create new plan with status=draft
```

## Entity Relationships

```
tenants (1) ──── (N) tax_plans           (tenant_id FK)
xpm_clients (1) ──── (N) tax_plans       (xpm_client_id FK)
xero_connections (1) ──── (N) tax_plans   (xero_connection_id FK, nullable)

tax_plans (1) ──── (N) tax_scenarios      (tax_plan_id FK, CASCADE)
tax_plans (1) ──── (N) tax_plan_messages  (tax_plan_id FK, CASCADE)

```
